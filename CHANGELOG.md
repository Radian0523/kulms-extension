# Changelog

## v1.7.3

- 「取組中」の課題が「完了済み」に誤分類されるバグを修正
  - Sakai API の `userSubmission` は下書き保存でも `true` になるため、`submitted && !draft` で判定するよう修正

## v1.7.0

- サイドバー色分けスタイル選択: ボーダー / 背景 / 太バー + 背景 の3モードから選択可能
- `color-mix()` による背景色の自動計算（カスタムカラーに自動追従）
- `is-current-site` との色重ねをモード別に正しく処理
- 設定画面にフィードバックメッセージを追加（個人開発の紹介）
- Ko-fi 応援リンクに説明文を追加
- 教科書パネルに各科目のシラバスページへのリンクを追加
- シラバスHTMLパーサーの `<style>` / `<script>` ブロック除去を修正
- 「デフォルトに戻す」ボタンが数値・カラー・セレクト入力にも適用されるよう修正
- デフォルト設定変更: メモ機能ON、色分けスタイルを「太バー + 背景」に
- 設定画面の「変更はページ再読み込み後に反映」をボタン行の直下に移動

## v1.6.0

- テーマ切り替え機能を削除（デフォルト/ダーク/セピア/ブルー）
- i18n言語上書き: `_locales` を `web_accessible_resources` に追加し、コンテンツスクリプトからの `fetch` を確実に

## v1.5.0

- 多言語対応 (i18n): 日本語/英語の自動切り替え + 設定から手動切り替え
- クイズ/テスト対応: Sakai `sam_pub` APIからクイズを取得し課題一覧に統合表示
- 緊急度閾値カスタマイズ: 緊急（時間）・注意（日）・余裕（日）を設定から変更可能
- 緊急度カラーカスタマイズ: 4色をカラーピッカーで変更、CSS変数で全体に反映
- 遅延提出セクション: 期限切れの未提出課題を独立セクションで表示
- メモ機能強化: コース選択・締切日時の設定が可能に、締切付きメモは緊急度セクションに統合
- 残り時間に分表示を追加（「残り3日2時間15分」形式）
- チェック状態を `entityId` ベースに改善（課題名変更時の状態消失を防止）
- 削除機能: 課題・メモの削除（2クリック確認）、「削除済み」セクションに移動、30日後に自動完全削除、復元可能
- 設定UIをグループ化セクションに再構成（外観・パネル・サイドバー・コースページ・課題更新・緊急度・開発者）
- 提出状態の判定を個別API (`/direct/assignment/item/`) に変更し、正確な提出済み検出を実現

## v1.4.0

- 課題の自動フェッチ機能と更新間隔の設定を追加（デフォルト120秒）
- 課題提出ボタンのクリック検知で自動完了チェック + リフレッシュ
- 提出済み課題を「完了済み」セクションに統合、タブ復帰時に自動更新
- 期限切れ + 完了済みの課題を一覧から非表示にする
- 設定に「デフォルトに戻す」「全てON」「全てOFF」ボタンを追加
- 課題リンクをSakai APIフォールバックで確実にポータル内遷移に修正
- ピン留めソートを科目名整理に依存しないよう修正

## v1.3.0

- KULMS+ にリネーム
