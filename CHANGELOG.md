# Changelog

## v1.9.5

- 教科書取得の精度向上: 同名科目が複数ヒットした場合に教員名で絞り込み
  - `lectureCode` 検索結果に同名科目が複数あった場合、Sakai のサイト情報から取得した教員名で 1 件に絞り込む
  - Sakai サイト情報「サイト連絡先・メール」欄の教員名を遅延フェッチして利用 (常にフェッチするわけではない)
  - 教員名比較は空白を除去して `includes` 双方向で照合 (例: KULASIS「久門 尚史」⇔ Sakai「久門尚史」)
  - サイト情報ツールがない / 連絡先未設定の科目では従来どおり最初の候補を採用
  - 科目名検索 (lectureCode が取れない場合) でも同名複数ヒット時に教員名で絞り込み
- Chrome 版 / Safari 版の両方を同期

## v1.9.4

- 教科書取得の修正 (v1.9.3 のリグレッション対応)
  - `lectureCode` 検索で検索エンジンが意図しない科目を先頭に返すケースに対応
    - 例: `6207000` 検索 → 1 件目に「義肢装具学概論」, 2 件目に正しい「電気電磁回路」
  - 検索結果の先頭を無条件に採用する挙動を廃止し、Sakai 側の科目名と一致するものを抽出
    - 完全一致 → 部分一致 の順で照合
  - `lectureCode` 検索で一致しなかった場合は科目名検索にフォールバック
  - 教科書キャッシュキーを `kulms-textbooks-v3` に更新（v1.9.3 で誤キャッシュされた内容を破棄）
  - Chrome 版 / Safari 版の両方を同期

## v1.9.3

- 教科書取得を site.id 由来の `lectureCode` ベースに変更 (#9, Thanks @AttoCat!)
  - Sakai API の各 `site.id` (例: `2026-888-H352-001`) から `lectureCode` (`H352001`) を抽出
  - `https://www.k.kyoto-u.ac.jp/external/open_syllabus/search` に `lectureCode` を渡し、検索結果の先頭を採用
  - `lectureCode` が取得できない場合は従来の科目名検索にフォールバック
  - 同名科目の誤認識問題 (#7) を解決
  - 教科書キャッシュキーを `kulms-textbooks-v2` に更新（既存キャッシュとの衝突回避）
  - Chrome 版 / Safari 版の両方を同期

## v1.9.1

- 授業中ハイライトに NEXT バッジを追加
  - 休み時間中は次の時限の科目に青い「NEXT」バッジを表示
  - 1限は開始15分前（8:30）から NEXT バッジを表示
  - 5限終了後（18:15〜）は何も表示しない
  - NOW（赤・点滅）と NEXT（青・静止）で視覚的に区別

## v1.9.0

## v1.8.1

- 提出状態の自動判定をオン/オフできる設定を追加（設定 → 課題更新セクション）
  - ON（デフォルト）: 提出済み・評定済みの課題を自動で完了済みに分類（従来動作）
  - OFF: 手動チェックのみで完了判定。提出ボタンクリック時の自動チェックも無効化
- 個別APIの提出判定で `submitted` ではなく `userSubmission` を参照するよう修正

  - 完了済みセクションはデフォルトで折りたたみ
